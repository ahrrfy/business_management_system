## ١. ملخّص تنفيذي

النظام بلغ نضجاً معمارياً جيداً: العمود الفقري (مخزون/دفتر/أموال/ذرّية) مُختبَر، وأغلب الشرائح الجوهرية مدموجة خلفيةً وواجهةً. **لكن المراجعة كشفت طبقة أمان مفقودة بالكامل تجعل النظام غير صالح لإدخال بيانات حقيقية بعد**:

- 🔴 **لا تفويض أدوار (RBAC) في الخادم إطلاقاً** — أي مسجّل (كاشير) يستطيع حذف منتجات، تعديل أسعار، إنشاء مشتريات، تحويل مخزون، سحب تقارير الذمم.
- 🔴 **تسريب التكلفة** (`costTotal`/`unitCost`/`costPrice`) لأي مسجّل ⇒ كشف هامش الربح.
- 🔴 **IDOR عبر `branchId`** — كاشير فرع المبيعات يرى فواتير وكشوف الفرع الرئيسي.
- 🔴 **`auditLogs` معرَّف في المخطّط ولا يُكتب فيه سطر** ⇒ صفر مساءلة.
- 🔴 **JWT صلاحيته سنة بلا إبطال** + **لا rate-limit على `login`** مع كلمة مرور افتراضية منشورة في `CLAUDE.md`.
- 🔴 **خطر فقد البيانات** (حاوية MySQL بلا named volume) — **قيد المعالجة الآن من جلسة `infra-prod`**.

**الخلاصة:** قبل أي بيانات حقيقية، أغلِق الأمن (RBAC + audit + auth hardening) والنسخ الاحتياطي، ثم استكمِل المفقود الحقيقي (الموردون CRUD، إدارة المستخدمين، عروض الأسعار، طباعة الملصقات).

## ٢. 🔴 حرج/عاجل (critical + high) — ١٤ بنداً لا تتجاوزها أي جلسة

### ٢.١ ✅ [critical] غياب فحوصات الأدوار في الخادم
- **الملف:** `server/trpc.ts` (فيه `protectedProcedure` + `adminProcedure` **فقط**).
- **الأثر:** الأدوار `cashier|manager|warehouse` معرَّفة في `users.role` لكنها لا تُستعمل في أي middleware ⇒ كاشير = صلاحيات كاملة. خرق فصل واجبات تام.
- **الإصلاح:** أضِف في `trpc.ts`:
  ```ts
  const requireRole = (...allowed: string[]) => t.middleware(async ({ ctx, next }) => {
    if (!ctx.user) throw new TRPCError({ code: "UNAUTHORIZED", message: UNAUTHED_ERR_MSG });
    if (ctx.user.role !== "admin" && !allowed.includes(ctx.user.role))
      throw new TRPCError({ code: "FORBIDDEN", message: "صلاحيات غير كافية" });
    return next({ ctx: { ...ctx, user: ctx.user } });
  });
  export const managerProcedure   = t.procedure.use(requireRole("manager"));
  export const cashierProcedure   = t.procedure.use(requireRole("cashier", "manager"));
  export const warehouseProcedure = t.procedure.use(requireRole("warehouse", "manager"));
  ```
  ثم طبّق: `manager` على `catalog.create/updateProduct`, `purchase.createOrder`, `return.create`, `reports.*`, `customer/supplier.create/update`; `warehouse|manager` على `inventory.transfer/adjust`, `purchase.receive`, `workOrder.start/markReady`; `cashier+` على `sale.*`, `shift.open`, `expense.create`.

### ٢.٢ ✅ [critical] تسريب التكلفة لكل مسجّل
- **الملف:** `server/routers/saleRouter.ts:106` (`costTotal`), `:127` (`unitCost`)؛ `workOrderRouter.ts` (`materialsCost`/`laborCost`/`unitCost`)؛ `catalogRouter.ts` (`catalog.forPurchase` → `costPrice`).
- **الأثر:** الكاشير يعرف هامش الربح بفتح أداة المطوّر.
- **الإصلاح:**
  ```ts
  // trpc.ts
  export const canSeeCost = (role: string) => role === "admin" || role === "manager";
  ```
  - `catalog.forPurchase` ⇐ `managerProcedure` (شاشة المشتريات بأكملها لمدير).
  - `sales.get/list` و`workOrders.get/list`: ابنِ الاستجابة شرطياً — `if (!canSeeCost(ctx.user.role)) delete row.costTotal/unitCost`.

### ٢.٣ ✅ [critical] IDOR عبر `branchId`
- **الملف:** `saleRouter.ts:86` (get/list)، + purchases/returns/expenses/reports/inventory/customers/suppliers (لا فلتر `branchId` ولا فحص ملكية).
- **الأثر:** كاشير SALES يستعرض بـid متسلسل كل عمليات MAIN وكل كشوف العملاء.
- **الإصلاح:**
  ```ts
  // trpc.ts
  export const branchScopedProcedure = protectedProcedure.use(({ ctx, next }) => {
    const elevated = ctx.user.role === "admin" || ctx.user.role === "manager";
    return next({ ctx: { ...ctx, scopedBranchId: elevated ? null : ctx.user.branchId } });
  });
  ```
  ثم في كل query: `.where(ctx.scopedBranchId ? eq(t.branchId, ctx.scopedBranchId) : undefined)`. في `get`: بعد الجلب `if (ctx.scopedBranchId && row.branchId !== ctx.scopedBranchId) throw NOT_FOUND` (لا FORBIDDEN — لمنع كشف الوجود). في `sale.create` أزل `ctx.user.branchId ?? input.branchId` ⇒ افرض فرع الكاشير.

### ٢.٤ ✅ [high] JWT صلاحيته سنة بلا revocation
- **الملف:** `server/auth/session.ts:23,37` (`ONE_YEAR_MS`)، `authRouter.ts:37-38,44`.
- **الأثر:** سرقة الكوكي = اختراق سنة. المدير لا يستطيع طرد موظف غادر؛ `logout` يمسح الكوكي فقط والتوكن يبقى صالحاً.
- **الإصلاح:** أضِف `users.sessionsValidFrom DATETIME DEFAULT NOW()`؛ في `getUserFromRequest` ارفض إن `payload.iat*1000 < sessionsValidFrom`؛ اضبط الافتراضي إلى ٨–١٢ساعة (تذكّرني = ٣٠ يوماً كحد أقصى — **لا سنة**)؛ عند تغيير كلمة المرور/تعطيل/تغيير دور ⇒ `sessionsValidFrom = NOW()`؛ أضِف `auth.revokeMySessions` و`users.revokeSessions(userId)` للأدمن.

### ٢.٥ ✅ [high] لا rate-limit/lockout على `login` مع كلمة مرور افتراضية مكشوفة
- **الملف:** `authRouter.ts:22`. `admin@alroya.local`/`Admin@12345` منشورة في `CLAUDE.md §٣`.
- **الإصلاح:** middleware على إجراء `login` (per-IP ٥/١٥د + per-email ١٠/ساعة ⇒ `TOO_MANY_REQUESTS`)؛ حقول `users.failedLoginAttempts`/`lockedUntil`؛ اطرد كلمة المرور الافتراضية إلى `ADMIN_INITIAL_PASSWORD` من env وأزِلها من `CLAUDE.md`.

### ٢.٨ ✅ [high] `verifyPassword`/`login` تسريب توقيت ⇒ تعداد مستخدمين

### ٢.٩ ✅ [high] `register` سباق إنشاء مكرّر — لا UNIQUE ولا `withTx`
